# Solicitar uma versão {#request-a-release}

## Pré-requisitos {#prerequisites}

* Você tem a função de **Gerente de versões** — consulte <!-- broken link[User roles](../teams/user-roles.md) -->
* Seu aplicativo está integrado — consulte [Integrar seu aplicativo](../applications/onboard-your-application.md)

>[!TIP]
>
>Antes de solicitar uma versão, reveja <!--[Releases and cross-team feature groups](releases-and-cross-team-feature-groups.md)-->. Um grupo de recursos entre equipes pode atender às suas necessidades com menos sobrecarga, é de autoatendimento e oferece suporte ao direcionamento de público-alvo mais avançado.

## Enviar uma solicitação de suporte {#submit}

A criação da versão não é de autoatendimento. Entre em contato com o suporte a Sinalizadores para solicitar uma nova versão. Forneça as seguintes informações:

| Campo | Descrição |
|---|---|
| **Nome** | Um nome exclusivo para a versão. Não inclua espaços no nome da versão. |
| **Equipe** | A equipe que será proprietária da versão. |
| **Ambiente** | O ambiente no qual a versão deve ser criada (Preparo ou Produção). |
| **Objetivo** | Uma breve descrição sobre o que esta versão implementará e por quê. |
| **Aplicativos** | O aplicativo ou aplicativos a serem incluídos na versão. |
| **Duração** | Por quanto tempo você precisa da versão para permanecer ativa. As versões devem ser fechadas ou colocadas na linha de base dentro de três meses. Se precisar de mais tempo, forneça uma justificativa de negócios. |

>[!NOTE]
>
>O número de versões ativas em um determinado momento é limitado. Planeje incluir na linha de base ou feche seu lançamento em três meses para liberar capacidade para outras equipes.

## Depois que a versão for criada {#after}

Depois de receber a confirmação de que a versão foi criada, faça logon no console e conclua a configuração da versão. Consulte [Fluxo de trabalho de versão de ponta a ponta](release-workflow-end-to-end.md) para obter a sequência completa de etapas.
