# Tipos de atividade do [!DNL Target]

Baixe um PDF interativo que descreve os diferentes tipos de atividades no [!DNL Adobe Target].

>[!NOTE]
>
>Para ter a melhor experiência e compartilhar com outras pessoas, baixe o [PDF interativo Guia de atividades do Adobe Target](/help/main/assets/activities_guide_82817.pdf).
>
>Este artigo não contém informações sobre [[!UICONTROL Atividades do Recommendations]](/help/main/c-recommendations/recommendations.md). No entanto, você pode incluir recomendações nas atividades de [!UICONTROL Teste A/B], [!UICONTROL Alocação automática], [!UICONTROL Direcionamento automático] e [!UICONTROL Direcionamento de experiência] (XT). Para obter mais informações, consulte [Recomendações como oferta](/help/main/c-recommendations/recommendations-as-an-offer.md). Essa funcionalidade exige uma [licença do Target Premium](/help/main/c-intro/intro.md#premium).

## O que ele faz? {#section_4ECAACC68723402EB3649033190E1BBC}

| Tipo de atividade | Detalhes |
|--- |--- |
| [!UICONTROL Teste A/B Manual]<P>![Ícone](/help/main/c-activities/assets/icon_ab.png) | Compara duas ou mais experiências para ver qual melhora mais as conversões durante um período de teste pré-especificado.<P>Para obter mais informações, consulte [Teste A/B](/help/main/c-activities/t-test-ab/test-ab.md). |
| [!UICONTROL Alocação automática]<P>![Ícone de Alocação automática](/help/main/c-activities/assets/icon_auto_allocate.png) | Identifica um vencedor entre duas ou mais experiências e redireciona tráfego para a experiência vencedora, aumentando a conversão à medida que o teste aprende enquanto é executado.<P>Para obter mais informações, consulte [Alocação automática](/help/main/c-activities/automated-traffic-allocation/automated-traffic-allocation.md). |
| [!UICONTROL Direcionamento automático]<P>![Ícone de AT](/help/main/c-activities/assets/icon_auto_target.png) | Usa aprendizagem de máquina avançada para personalizar conteúdo e impulsionar as conversões, identificando várias experiências de alta performance definidas por profissionais de marketing e depois apresentando a experiência mais adequada para visitantes com base no perfil individual do cliente e no comportamento anterior de visitantes com perfis similares.<P>Para obter mais informações, consulte [Direcionamento automático para experiências personalizadas](/help/main/c-activities/auto-target/auto-target-to-optimize.md). |
| [!UICONTROL Automated Personalization] (AP)<P>![Ícone de AP](/help/main/c-activities/assets/icon_ap.png) | Usa aprendizagem de máquina avançada para personalizar conteúdo e impulsionar as conversões, combinando ofertas e mensagens específicas e então correspondendo diferentes variações de ofertas com visitantes, com base nos perfis individuais de clientes.<P>Para obter mais informações, consulte [Automated Personalization](/help/main/c-activities/t-automated-personalization/automated-personalization.md). |
| [!UICONTROL Teste multivariado] (MVT)<P>![Ícone de MVT](/help/main/c-activities/assets/icon_mvt.png) | Compara combinações de ofertas entre elementos em uma página para determinar qual combinação tem o melhor desempenho para um público-alvo específico. Também identifica qual elemento da página melhora mais conversões durante um período de teste pré-especificado.<P>Para obter mais informações, consulte [Teste multivariado](/help/main/c-activities/c-multivariate-testing/multivariate-testing.md). |
| Direcionamento de experiência (XT)<P>![Ícone de XT](/help/main/c-activities/assets/icon_xt.png) | Fornece conteúdo para um público-alvo específico baseado em um conjunto de regras e critérios definidos por um profissional de marketing.<P>Para obter mais informações, consulte [Direcionamento de experiência](/help/main/c-activities/t-experience-target/experience-target.md). |

## Por que você está usando esse tipo de atividade? {#section_46A70DD7CE3448749E635DDF5EAFC131}

| Tipo de atividade | Motivo |
|--- |--- |
| [!UICONTROL Teste A/B Manual] | Um experimento altamente controlado com medições de tráfego, dividido em porcentagens em vez de uma regra, permitindo analisar os dados do teste, obter insights sobre seu público-alvo e determinar qual experiência tem o melhor desempenho. |
| [!UICONTROL Alocação automática] | Um meio de identificar uma experiência vencedora e ajustando a alocação de tráfego para entregá-la aos visitantes o mais rápido possível, oferecendo uma probabilidade maior e mais rápida de conversão. |
| [!UICONTROL Direcionamento automático] | Um meio de identificar vencedores entre várias experiências e então entregar a experiência mais apropriada para visitantes específicos. O direcionamento se adapta com o tempo conforme os interesses dos visitantes mudam, pois o algoritmo prevê a tendência de conversão de um visitante em uma determinada experiência em um determinado tempo. |
| [!UICONTROL Automated Personalization] (AP) | Um meio de personalizar um conjunto e ofertas (criadas ou pré-definidas, em elemento em uma única página ou em várias páginas) e entregar combinações de ofertas que funcionam melhor para atrair visitantes específicos. |
| [!UICONTROL Multivariate Testing] (MVT) | Uma maneira de exibir várias ofertas em vários elementos e, em seguida, testar as experiências exclusivas resultantes simultaneamente em relação a uma meta específica, o que ajuda a determinar qual variação de elemento é a mais bem-sucedida. Uma atividade de MVT também pode revelar quais elementos têm mais impacto positivo ou negativo na interação de visitantes. |
| [!UICONTROL Direcionamento de experiência] (XT) | Um meio de direcionar conteúdo específico para um público-alvo específico com base em um conjunto de regras de alocação definidas. |

## Que tipo de profissional de marketing deve usar o tipo de atividade? {#section_A843D663D3E543FFB1A594266B560395}

| Tipo de atividade | O profissional de marketing |
|--- |--- |
| [!UICONTROL Teste A/B Manual] | Possui conhecimento de estatísticas.<P>Tem tempo para aguardar até o fim do período de teste para analisar resultados. |
| [!UICONTROL Alocação automática] | Tem um prazo curto.<P>Deve identificar a melhor experiência e entregá-la rapidamente.<P>Quer ser capaz de “espiar” os resultados conforme o teste é executado. |
| [!UICONTROL Direcionamento automático] | Possui várias experiências elegíveis.<P>Quer corresponder experiências com visitantes específicos em épocas ideais com base em seus perfis dinâmicos e em alteração. |
| [!UICONTROL Automated Personalization] (AP) | Possui uma ou mais ofertas.<P>Deseja criar combinações de ofertas que proporcionem experiências personalizadas ideais para visitantes específicos em vários perfis e comportamentos únicos. |
| [!UICONTROL Multivariate Testing] (MVT) | Possui conhecimento de estatísticas.<P>Possui uma ou mais ofertas.<P>Quer analisar tendências de conversão relacionadas às interações com elementos da página. |
| [!UICONTROL Direcionamento de experiência] (XT) | Deve entregar uma experiência específica ou um pedaço de conteúdo para um público-alvo específico. |

## Detalhes estatísticos {#section_22CF2D07DB054505AB5EC702B99A5BB0}

| Tipo de atividade | Detalhes |
|--- |--- |
| [!UICONTROL Teste A/B Manual] | O teste compara cada experiência proposta com uma experiência de controle e depois classifica o desempenho de todas elas, identificando tanto uma experiência vencedora quanto uma experiência perdedora quando comparada ao controle. |
| [!UICONTROL Alocação automática] | O teste produz uma garantia estatística no vencedor verdadeiro imediatamente e então direciona mais tráfego para públicos-alvo que possuem uma probabilidade de conversão maior com a experiência vencedora. |
| [!UICONTROL Direcionamento automático] | O mecanismo de otimização identifica o público-alvo relevante para cada experiência, mostrando aumentos e diminuições ao longo do tempo e antes de determinar qual experiência será entregue a qual visitante. O mecanismo de otimização é informado por conversões, segmentos, parâmetros e scripts de perfil. A partir daí, o mecanismo escolhe automaticamente qual algoritmo usar para gerar uma elevação e uma taxa de conversão mais altas. |
| [!UICONTROL Automated Personalization] (AP) | O mecanismo de otimização constantemente ajusta quais experiências são entregues para quais visitantes com base no comportamento de um novo visitante e em comportamentos anteriores de visitantes semelhantes, com o desempenho de uma oferta sendo medida contra grupos de controle simultaneamente. |
| [!UICONTROL Multivariate Testing] (MVT) | O teste ajuda a descobrir a influência relativa que elementos específicos possuem em uma conversão. |
| [!UICONTROL Direcionamento de experiência] (XT) | O método define regras que direcionam ou uma experiência específica ou uma pedaço de conteúdo específico para um determinado público-alvo. Os clientes podem fazer atualizações no nível da experiência. |

## Benefícios e considerações {#section_56C46ABEF7B945DDA0C1E6D714377123}

| Tipo de atividade | Benefícios | Considerações |
|--- |--- |--- |
| [!UICONTROL Teste A/B Manual] | Os testes A/B ajudam a obter uma compreensão completa do desempenho de cada experiência, além de apenas qual experiência tem o melhor desempenho. | Em um [!UICONTROL Teste A/B], se você observar os resultados do teste antes de atingir o tamanho da amostra, corre o risco de confiar em resultados imprecisos (não é possível &quot;espiar&quot;mais cedo!).<P>Ao contrário da [!UICONTROL Alocação automática], em um teste A/B, a distribuição de tráfego permanece fixa mesmo após você reconhecer que algumas experiências estejam com desempenho melhor que outras.<P>Para obter informações sobre as práticas recomendadas para atividades de [!UICONTROL Teste A/B], consulte [Por quanto tempo você deve executar um Teste A/B](/help/main/c-activities/t-test-ab/sample-size-determination.md) e [Dez erros comuns em testes A/B e como evitá-los](/help/main/c-activities/t-test-ab/common-ab-testing-pitfalls.md). |
| [!UICONTROL Alocação automática] | A [!UICONTROL Alocação automática] reduz o custo de um teste A/B típico porque tem uma taxa de conversão geral mais alta do que um teste A/B manual. A taxa de conversão é maior porque a [!UICONTROL Alocação automática] envia mais tráfego para a experiência de maior desempenho, o que significa que você pode obter os benefícios dessa experiência vencedora antes do final do período de teste (você pode espiar!). | [!UICONTROL Alocação automática] identifica o vencedor, mas não diferencia os perdedores. Se você precisar saber o desempenho de cada experiência, o teste A/B é preferível.<P>O recurso [!UICONTROL Alocação automática] funciona somente com uma configuração de métrica avançada, que é &quot;Aumentar contagem e manter o usuário na atividade&quot;. Se não quiser contar as conversões repetidas, é melhor usar o teste A/B. |
| [!UICONTROL Direcionamento automático] | Com o [!UICONTROL Direcionamento automático], o aprendizado de máquina é aplicado a qualquer tipo de experiência, incluindo experiências de várias páginas. Uma atividade de [!UICONTROL Direcionamento automático] também permite que você ganhe o valor de [!UICONTROL Automated Personalization] ao usar o fluxo de trabalho familiar de teste A/B. | Com o [!UICONTROL Direcionamento automático], se você quiser alterar o conteúdo de suas ofertas com frequência, o algoritmo precisará de tempo suficiente após cada alteração para explorar o que aprende e entregar esse conteúdo aos visitantes certos. |
| [!UICONTROL Automated Personalization] (AP) | Com o [!UICONTROL Automated Personalization], você pode coletar todas as suas ofertas em um único local e o algoritmo determina a melhor combinação de ofertas. Você não precisa especificar ou construir experiências individuais. [!UICONTROL O Automated Personalization] usa os mesmos algoritmos de aprendizado de máquina que [!UICONTROL Direcionamento automático]. | Ao combinar várias ofertas, uma explosão de combinações ocorre, resultando na necessidade de uma quantidade significativa de tráfego. O algoritmo [!UICONTROL Automated Personalization] leva em conta muitos fatores; portanto, requer a maior quantidade de tráfego.<P>O [!UICONTROL Automated Personalization] não pode consumir relatórios no [Analytics for Target](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T). |
| [!UICONTROL Multivariate Testing] (MVT) | Com o [!UICONTROL Multivariate Testing], você pode testar vários elementos simultaneamente. | Um [!UICONTROL Teste multivariado] é demorado e, devido às várias variáveis em jogo, ele não produz necessariamente uma experiência vencedora com confiança.<P>Geralmente é difícil alcançar a quantidade de tráfego necessária para completar o teste. Como todos experimentos de [!UICONTROL Teste multivariado] são totalmente fatoriais, muitos elementos em alteração ao mesmo tempo podem adicionar rapidamente até muitas combinações possíveis que devem ser testadas.<P>Mesmo um site com tráfego relativamente alto pode ter problemas em completar um teste com mais de 25 combinações em um tempo razoável. |
| [!UICONTROL Direcionamento de experiência] (XT) | Com o [!UICONTROL Direcionamento de experiência], você pode agir rapidamente com base em insights deduzidos de qualquer resultado de atividade.<P>Por exemplo, se você realizar um teste A/B no qual o desafiante não obteve desempenho melhor que o controle, mas os resultados indicam que um segmento específico de visitantes converteu quatro vezes mais com desafiante do que com o controle, você pode usar o [!UICONTROL Direcionamento de experiência] para direcionar a experiência desafiante para esse segmento específico. | [!UICONTROL O Direcionamento de experiência] não permite que você controle a divisão de porcentagem de uma experiência entre vários públicos. |
