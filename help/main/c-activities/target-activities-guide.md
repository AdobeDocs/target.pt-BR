# [!DNL Target] tipos de atividades

Baixe um PDF interativo que descreve os diferentes tipos de atividades no [!DNL Adobe Target].

>[!NOTE]
>
>Observação: para ter a melhor experiência e compartilhar com outras pessoas, baixe o PDF interativo [Guia de atividades do Adobe Target](/help/main/assets/activities_guide_82817.pdf).

No entanto, você pode incluir recomendações no [!UICONTROL Teste A/B], [!UICONTROL Alocação automática], [!UICONTROL Direcionamento automático], e [!UICONTROL Direcionamento de experiência] (XT) Atividades. ## O que ele faz? {#section_4ECAACC68723402EB3649033190E1BBC}

| Tipo de atividade | Detalhes |
|--- |--- |
| Manual [!UICONTROL Teste A/B]<P>![ícone](/help/main/c-activities/assets/icon_ab.png) | Compara duas ou mais experiências para ver qual experiência melhora mais conversões durante um período de teste pré-especificado.<P>Para obter mais informações, consulte [Teste A/B](/help/main/c-activities/t-test-ab/test-ab.md). |
| [!UICONTROL Alocação automática]<P>![Ícone Alocação automática](/help/main/c-activities/assets/icon_auto_allocate.png) | Identifica um vencedor entre duas ou mais experiências e redireciona o tráfego para a experiência vencedora, aumentando a conversão conforme o teste é executado e aprendido.<P>Para obter mais informações, consulte [Alocação automática](/help/main/c-activities/automated-traffic-allocation/automated-traffic-allocation.md). |
| [!UICONTROL Direcionamento automático]<P>![Ícone AT](/help/main/c-activities/assets/icon_auto_target.png) | Usa aprendizagem de máquina avançada para personalizar conteúdo identificando várias experiências de alta performance definidas por profissionais de marketing e depois apresentando a experiência mais personalizada para visitantes com base no perfil individual do cliente e no comportamento de visitantes anteriores com perfil similares. <P>Para obter mais informações, consulte [Direcionamento automático para experiências personalizadas](/help/main/c-activities/auto-target/auto-target-to-optimize.md). |
| [!UICONTROL Automated Personalization] (AP)<P>![Ícone AP](/help/main/c-activities/assets/icon_ap.png) | Usa aprendizagem de máquina avançada para personalizar conteúdo e realizar conversões combinando ofertas e mensagens específicas e então correspondendo diferentes variações de ofertas com visitantes, com base no perfil individual do cliente. <P>Para obter mais informações, consulte [Automated Personalization](/help/main/c-activities/t-automated-personalization/automated-personalization.md). |
| [!UICONTROL Teste multivariado] (MVT)<P>![Ícone MVT](/help/main/c-activities/assets/icon_mvt.png) | Compara combinações de ofertas entre elementos em uma página para ver qual combinação de ofertas tem o melhor desempenho para um público-alvo específico. Também identifica qual elemento da página melhora mais conversões durante um período de teste pré-especificado. <P>Para obter mais informações, consulte [Teste multivariado](/help/main/c-activities/c-multivariate-testing/multivariate-testing.md). |
| Direcionamento de experiência (XT)<P>![Ícone XT](/help/main/c-activities/assets/icon_xt.png) | Leva conteúdo para um público-alvo específico baseado em um conjunto de regras e critérios definidos pelo profissional de marketing.<P>Para obter mais informações, consulte [Direcionamento de experiência](/help/main/c-activities/t-experience-target/experience-target.md). |

## Por que você está usando esse tipo de atividade? {#section_46A70DD7CE3448749E635DDF5EAFC131}

| Tipo de atividade | Motivo |
|--- |--- |
| Manual [!UICONTROL Teste A/B] | Um experimento altamente controlado com medida de tráfego, dividido em porcentagens em vez de uma regra, permitindo que você analise os dados do teste, recolha insights sobre seu público-alvo e determine qual experiência tem a melhor performance. |
| [!UICONTROL Alocação automática] | Uma maneira de identificar uma experiência vencedora e ajustar a alocação de tráfego para fornecer a experiência vencedora aos visitantes o mais rápido possível, oferecendo suporte a uma probabilidade mais rápida e alta de conversão. |
| [!UICONTROL Direcionamento automático] | Um meio de identificar vencedores entre várias experiências e então entregar a experiência mais apropriada para visitantes específicos. O direcionamento se adapta com o tempo conforme os interesses dos visitantes mudam, pois o algoritmo prevê a tendência de conversão de um visitante em uma determinada experiência em um determinado tempo. |
| [!UICONTROL Automated Personalization] (AP) | Um meio de personalizar um conjunto e ofertas (criadas ou pré-definidas, em elemento em uma única página ou em várias páginas) e entregar combinações de ofertas que funcionam melhor para atrair visitantes específicos. |
| [!UICONTROL Multivariate Testing] (MVT) | Uma maneira de exibir várias ofertas em vários elementos e, em seguida, testar as experiências exclusivas resultantes simultaneamente em relação a uma meta específica, o que ajuda a determinar qual variação de elemento é a mais bem-sucedida. Uma atividade de MVT também pode revelar quais elementos têm mais impacto positivo ou negativo na interação do visitante. |
| [!UICONTROL Direcionamento de experiência] (XT) | Uma maneira de direcionar conteúdo específico a um público-alvo específico com base em um conjunto de regras de alocação definidas. |

## Que tipo de profissional de marketing deve usar o tipo de atividade? {#section_A843D663D3E543FFB1A594266B560395}

| Tipo de atividade | O profissional de marketing |
|--- |--- |
| Manual [!UICONTROL Teste A/B] | Possui conhecimento de estatísticas.<P>Tem tempo para aguardar até o fim do período de teste para analisar resultados. |
| [!UICONTROL Alocação automática] | Tem um prazo curto.<P>Deve identificar a melhor experiência e fornecer rapidamente.<P>Quer ser capaz de &quot;espiar&quot; os resultados conforme o teste é executado. |
| [!UICONTROL Direcionamento automático] | Possui várias experiências elegíveis.<P>Quer corresponder experiências com visitantes específicos em épocas ideais com base em seus perfis dinâmicos e em alteração. |
| [!UICONTROL Automated Personalization] (AP) | Possui uma ou mais ofertas.<P>Quer criar combinações de ofertas que produzem experiências personalizadas ideais para visitantes específicos em vários perfis e comportamentos únicos. |
| [!UICONTROL Multivariate Testing] (MVT) | Possui conhecimento de estatísticas.<P>Possui uma ou mais ofertas.<P>Quer analisar tendências de conversão relacionadas às interações com elementos da página. |
| [!UICONTROL Direcionamento de experiência] (XT) | Deve fornecer uma experiência ou conteúdo específico a um público-alvo específico. |

## Detalhes estatísticos {#section_22CF2D07DB054505AB5EC702B99A5BB0}

| Tipo de atividade | Detalhes |
|--- |--- |
| Manual [!UICONTROL Teste A/B] | O teste compara cada experiência desafiante a uma experiência de controle e, em seguida, classifica o desempenho de todas as experiências, identificando uma experiência vencedora e uma experiência perdedora em comparação ao controle. |
| [!UICONTROL Alocação automática] | O teste produz uma garantia estatística em um verdadeiro vencedor imediatamente e então direciona mais tráfego para públicos-alvo que possuem uma probabilidade de conversão maior com a experiência vencedora. |
| [!UICONTROL Direcionamento automático] | O mecanismo de otimização identifica o público-alvo relevante para cada experiência, mostrando aumentos e reduções no aumento ao longo do tempo e antes de determinar qual experiência fornecer para cada visitante. O mecanismo de otimização é informado por conversões, segmentos, parâmetros e scripts de perfil. A partir daí, o mecanismo escolhe automaticamente qual algoritmo usar para gerar um aumento e uma taxa de conversão mais altos. |
| [!UICONTROL Automated Personalization] (AP) | O mecanismo de otimização constantemente ajusta quais experiências são entregues para quais visitantes com base no comportamento de um novo visitante e comportamentos anteriores de visitantes semelhantes, com a performance de uma oferta sendo medida contra grupos de controle ao mesmo tempo. |
| [!UICONTROL Multivariate Testing] (MVT) | O teste ajuda a descobrir a influência relativa que elementos específicos possuem em conversão. |
| [!UICONTROL Direcionamento de experiência] (XT) | O método define regras que direcionam uma experiência específico ou uma parte específica de conteúdo para um público-alvo específico. Os clientes podem fazer atualizações no nível da experiência. |

## Benefícios e considerações {#section_56C46ABEF7B945DDA0C1E6D714377123}

| Tipo de atividade | Benefícios | Considerações |
|--- |--- |--- |
| Manual [!UICONTROL Teste A/B] | O repouso A/B ajuda você a entender completamente como cada experiência funciona, além de qual experiência tem o melhor desempenho. | Em um [!UICONTROL Teste A/B]Além disso, se você observar os resultados do teste antes de atingir o tamanho da amostra, corre o risco de confiar em resultados imprecisos (não é possível &quot;espiar&quot;mais cedo!).<P>Ao contrário [!UICONTROL Alocação automática]No entanto, em um teste A/B, a distribuição de tráfego permanece fixa mesmo depois de você reconhecer que algumas experiências estejam com desempenho melhor que outras.<P>Para obter informações sobre as práticas recomendadas para [!UICONTROL Teste A/B] atividades, consulte [Por quanto tempo você deve executar um teste A/B](/help/main/c-activities/t-test-ab/sample-size-determination.md) e [Dez erros comuns em testes A/B e como evitá-los](/help/main/c-activities/t-test-ab/common-ab-testing-pitfalls.md). |
| [!UICONTROL Alocação automática] | [!UICONTROL A alocação automática reduz o custo do teste A/B comum pois tem uma taxa de conversão maior no geral do que um teste A/B manual. ] A taxa de conversão é maior porque [!UICONTROL Alocação automática] O envia mais tráfego para a experiência de mais alto desempenho, o que significa que você pode obter os benefícios dessa experiência vencedora antes do final do período de teste (você pode espiar!). | [!UICONTROL A alocação automática identifica o vencedor mas não diferencia os perdedores. ] Se você precisar saber como cada experiência foi executada, o teste A/B é preferível.<P>A variável [!UICONTROL Alocação automática] O recurso funciona somente com uma configuração de métrica avançada, que é &quot;Aumentar contagem e manter o usuário na atividade&quot;. Se não quiser contar conversões repetidas, você deve usar o teste A/B. |
| [!UICONTROL Direcionamento automático] | Com [!UICONTROL Direcionamento automático], o aprendizado de máquina é aplicado a qualquer tipo de experiência, incluindo experiências de várias páginas. Um [!UICONTROL Direcionamento automático] A atividade também permite que você ganhe o valor de [!UICONTROL Automated Personalization] ao usar o fluxo de trabalho familiar do teste A/B. | Com [!UICONTROL Direcionamento automático]Além disso, se você quiser alterar o conteúdo de suas ofertas com frequência, o algoritmo precisará de tempo suficiente após cada alteração para explorar o que aprende e entregar esse conteúdo aos visitantes certos. |
| [!UICONTROL Automated Personalization] (AP) | Com [!UICONTROL Automated Personalization], você pode coletar todas as suas ofertas em um único local e o algoritmo determina a melhor combinação de ofertas. Você não precisa especificar ou construir experiências individuais. [!UICONTROL Automated Personalization] O usa os mesmos algoritmos de aprendizado de máquina do [!UICONTROL Direcionamento automático]. | Quando você combina várias ofertas, uma explosão de combinações ocorre, resultando na necessidade de uma quantia significativa de tráfego. A variável [!UICONTROL Automated Personalization] o algoritmo leva em conta muitos fatores; portanto, requer a maior quantidade de tráfego.<P>[!UICONTROL Automated Personalization] não pode consumir relatórios no [Analytics for Target](/help/main/c-integrating-target-with-mac/a4t/a4t.md) (A4T). |
| [!UICONTROL Multivariate Testing] (MVT) | Com [!UICONTROL Multivariate Testing], é possível testar vários elementos simultaneamente. | A [!UICONTROL Teste multivariado] O é demorado e, devido às várias variáveis em jogo, não produz necessariamente uma experiência vencedora com confiança.<P>Geralmente é desafiante alcançar a quantidade de tráfego necessária para completar o teste. Porque todos [!UICONTROL Teste multivariado] experimentos são totalmente fatoriais, muitos elementos em alteração ao mesmo tempo podem adicionar rapidamente para muitas combinações possíveis que devem ser testadas.<P>Mesmo um site com tráfego relativamente alto pode ter problemas em completar um teste com mais de 25 combinações em uma quantia de tempo praticável. |
| [!UICONTROL Direcionamento de experiência] (XT) | Com [!UICONTROL Direcionamento de experiência], você pode agir rapidamente com base nos insights deduzidos de qualquer resultado de atividade.<P>Por exemplo, se você realizar um teste A/B em que o desafiante não obteve desempenho melhor que o controle, mas os resultados indicam que um segmento específico de visitantes converteu quatro vezes mais com desafiante do que com o controle, você pode usar [!UICONTROL Direcionamento de experiência] para direcionar a experiência desafiante para esse segmento específico. | [!UICONTROL O direcionamento de experiência não permite que você controle a divisão de porcentagem de uma experiência entre vários públicos-alvo.] |
